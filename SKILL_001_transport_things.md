# 物品搬运与交付编排

本文档只提供规划规范，不直接控制机器人。读取本文档后，使用当前可用的 Scene、Navigation、Pick、Speech、Executor 和 Soma 能力生成并执行 RTDL。

## 规划前必须读取的状态

每次初始规划、用户 steer 和失败重规划前，都重新读取：

1. Soma 本体状态：底盘是否移动、所有夹爪位置与 `likely_holding`、数据 freshness。
2. Scene 环境状态：当前位置、所在房间/区域、附近物体，以及来源房间和目标房间的稳定 ID。
3. Executor/Pilot RTDL forest：正在运行、已完成、正在取消的具体 plan 和节点。

状态缺失或 stale 就视为未知，不能从旧对话猜测。`likely_holding=true` 只说明夹爪没有完全张开，不能单独证明拿着哪件物体；物体身份还要结合成功的 Pick 结果、当前任务记录和必要的观察。

## 标准阶段

1. **理解任务**：识别物体、来源位置、收件人和交付位置。信息不足时先询问，不猜房间或人物。
2. **检查当前状态**：判断机器人当前在哪、是否正在移动、夹爪是否已持物、当前是否已有冲突任务。
3. **解析来源**：先调用 Scene 查询房间或区域，取得稳定 ID；再调用 `goal_room` 得到可导航位姿。Scene 结果与导航参数有数据依赖，必须分轮规划。
4. **准备出发播报**：通过 `speech/speak` 使用车载 speaker 简短说明准备去哪里拿什么。
5. **前往来源**：调用 Navigation，并等待明确成功；不能在尚未抵达时提前 Pick。
6. **取物播报与抓取**：播报准备拿取物体；仅在确认夹爪当前没有目标物体时调用 Pick。Pick 后重新读取 Soma，并结合 Pick 结果确认夹爪不再完全张开。不要为了确认而丢下后再抓一次。
7. **持物播报**：确认拿到后，播报将把物体送往哪个目标。
8. **解析并前往目标**：通过 Scene 的稳定 ID 和 `goal_room` 获取目标位姿，再导航并等待成功。
9. **到达与交付**：播报已经到达并提醒用户取走物体。只有在能力和用户指令明确要求释放时才执行放置或张开夹爪；不得把“提醒取走”擅自解释为立即丢下。
10. **完成确认**：确认机器人位于目标房间、导航已结束，并且交付动作或用户确认已完成，然后再向用户报告完成。

## 用户中途修改任务

- 用户修改目的地或收件人时，先从 RTDL forest 找到当前冲突的导航或交付 plan，只取消这个 plan，并等待取消成功。
- 不取消无关的长期任务，例如 greet；也不重复发送同一个 cancel。
- 若机器人已经持有目标物体，保留物体，播报新的目的地，重新通过 Scene 解析目标后从“前往目标”阶段继续；禁止放下再抓一次。
- 若尚未 Pick，则可取消前往旧来源的导航，重新解析新的来源或物体后继续。
- 若用户改变要搬的物体，但夹爪可能已持有旧物体，先说明当前状态并请求明确的放置位置，不能静默丢弃。
- “我改变主意了”本身不等于停止全部任务。只修改与新指令冲突的分支，保留已经完成且仍有用的结果。

## 并行与取消规则

- 搬运主链路中有数据依赖的阶段使用 sequence；没有依赖且不争用资源的长期 skill 可以作为独立 RTDL tree 并行运行。
- `cancel all` 只在用户明确要求停止全部任务或紧急停止时使用。
- cancel 请求发出后，在收到取消确认前不要下发同一资源的新动作。
- steer 后重新规划时必须复用当前本体和环境事实，不能从头机械重放整个任务。

## 示例

用户说：“我在 315，请从餐厅拿一瓶水给我。”预期流程是：确认当前位置和夹爪状态 → Scene 解析餐厅 → 播报准备取水 → 导航到餐厅 → 播报准备拿水 → Pick 一次并验证持物 → 播报前往 315 → Scene 解析 315 → 导航 → 播报到达并提醒用户取水 → 确认交付。

执行途中用户说：“我改变主意了，把水给开阔区域的李老师。”若水已经拿到：只取消原目标导航 → 等待取消成功 → 保持夹爪持物 → 播报新目标 → Scene 解析开阔区域 → 导航 → 到达后播报并交付。不得取消 greet，不得放下水再抓一次。

## 禁止行为

- 不用对话历史代替 Soma、Scene 或 RTDL 的实时状态。
- 不猜房间坐标、稳定 ID、夹爪是否持物或导航是否完成。
- 不重复 Pick、重复 cancel、重复 speak 或重复提交相同任务。
- 不在取消尚未确认时向同一底盘或机械臂下发冲突动作。
- 不把内部工具结果或重复的最终答复刷给用户。
